# Decisions already made (do NOT relitigate)

- Per-version frozen baseline + intervention windows combined in V3.
- Migration is V2→V3 LIGHTWEIGHT. Backfill runs in app code, idempotent, gated by
  UserDefaults key `protocolFormula.v3BackfillCompleted` (Bool).
- Window backfill ordering: sort versions by `(shippedOn ASC, createdAt ASC)`.
- For each version i in sorted order, emit `InterventionWindow`:
    startedAt = version.shippedOn
    endedAt   = version[i+1].shippedOn  if a successor exists,
                else version.archivedAt if archived (clamp to >= shippedOn),
                else nil (active)
    phase     = .active if endedAt == nil
                .archived if version.archivedAt != nil and no successor
                .superseded otherwise
- Archived versions DO get windows (endedAt = archivedAt, clamped).
- `versionID` backfill on existing `StoredProtocolBaselineSnapshot`: assign to the
  CURRENT active version's id (only one snapshot row exists pre-V3 due to the
  delete-all-then-insert pattern at LocalDataRepository.swift:685–695).
- Stop the delete-all-then-insert pattern. Replace `saveBaselineSnapshot(_:)` with
  upsert keyed by `(versionID)`. Keep legacy no-arg signature working (treat as
  "save for currently-active version").
- New repo methods to add:
    func saveBaselineSnapshot(_ snapshot: ProtocolBaselineSnapshot) async throws
    func fetchBaselineSnapshot() async throws -> ProtocolBaselineSnapshot?           // newest, back-compat
    func fetchBaselineSnapshot(versionID: UUID) async throws -> ProtocolBaselineSnapshot?
    func fetchInterventionWindows() async throws -> [InterventionWindow]
    func saveInterventionWindow(_ window: InterventionWindow) async throws            // upsert by id
    func deleteInterventionWindow(id: UUID) async throws
- Add to RepositoryProtocols.swift; provide default no-op fallbacks for
  `MockLocalDataRepository` consistent with existing pattern at L216–L229.
- `ProtocolBaselineService.freezeBaseline(...)` — overload with `versionID: UUID`
  param. Old singleton signature stays, calls new one with active version id.
- `ProtocolFormulaCatalogService.upsertVersion` (~L196) — when a new version is
  created, transactionally:

- Window emission is also required when a version is archived: close its window
