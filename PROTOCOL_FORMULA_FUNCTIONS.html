UI Design

Changes shipped in the May 21, 2026 UI unification pass.

Protocol Home — Last Night layout

  • Removed duplicate metric grid. The standalone 4-tile grid (Deep / REM / Awake / Sleep for the single last night) that appeared between the hero card and the Impact vs Baseline section has been removed. The hero card's stage bar already shows the nightly breakdown visually; the second grid was identical in shape and created visual noise.
  • Impact vs Baseline expanded to 5 metrics. Awake time has been added alongside Restorative %, Deep, REM, and Total Sleep so the unified section covers everything the removed grid carried. Metric order: restorativePct → deep → rem → awake → duration.

Stage Bar colors

Previously Deep and REM both derived their color from the active version's color hex (e.g. purple for V3), making them visually indistinguishable. Colors are now:

StageColorRationale
Deep Version color (e.g. #C084FC for V3) Anchors the bar to the active formula's identity color.
REM #4ADE80 — fixed light green Distinct from Deep; green reads as restorative/positive.
Light #60A5FA at 50% opacity — fixed light blue Neutral-cool tone; clearly separate from Deep and REM.
Awake badColor (#F87171) at 80% opacity Unchanged — red conveys interruption.

Legend dots in StageBar use the same color tokens. Both colors are defined as static constants on StageBar (remColor, lightColor) in ProtocolFormulaAtoms.swift.

Design tokens used: ProtocolPalette.versionColor(hex:) for Deep, #4ADE80 for REM (same as ProtocolPalette.goodColor), #60A5FA for Light (same as ProtocolFormulaMetric.restorativePct color hex). No new palette entries were added — existing token values were reused.

Data Model

Formula versions

  • ProtocolFormulaVersion represents V1, V2, V3, and later formulas.
  • formulaText and components are immutable after a night log references the version.
  • Imported placeholders may be backfilled once, then become locked like normal versions.

Night logs

  • ProtocolNightLog is one row per sleepDateKey.
  • .taken contributes to formula impact rollups.
  • .skipped is visible in timeline/adherence UI but excluded from formula impact rollups.
  • .unknown means no row and is excluded from taken-vs-skipped calculations.

Frozen baseline

  • ProtocolBaselineSnapshot is frozen once before protocol tracking starts.
  • Window: 90 days before first protocol night.
  • Sample: up to 30 most recent qualifying nights.
  • Qualifying data quality: .detailedStages or .mixedSources.
  • Insufficient when fewer than 7 valid nights exist; no snapshot is stored when zero valid nights exist.

Screen Behavior

Home: Last night

  • Shows last available cached sleep session from the recent window.
  • Restorative hero uses the night's snapshot and the active/version color.
  • Deep, REM, awake, and sleep duration tiles compare the single night against frozen baseline means.

Home: Tonight

  • Mark taken saves a .taken log with selected formula version and add-ins.
  • Didn't take saves a .skipped log with no add-ins.
  • Saved state is durable because it is read back from fetchNightLog(forSleepDateKey:).
  • Retry uses the failed status, so skipped retries remain skipped retries.

Impact vs Baseline

  • Compares active formula version rollup against the user's frozen baseline.
  • Requires a sufficient baseline and at least 3 taken nights for the active version.
  • Every displayed delta is paired with: Observed, not causal -- your baseline isn't a control group.

All Metrics

  • Supports every ProtocolFormulaMetric.
  • Trend chart includes version phase shading, baseline line, selected-point crosshair, and tooltip.
  • Tooltip includes date, value, formula version, baseline value, and delta.

Timeline

  • Cards are created from stored protocol logs first.
  • Rollup metrics attach only when matching taken logs and cached sleep sessions exist.
  • Skipped nights appear in counts/heatmap but do not affect impact rollups.
  • The baseline anchor shows reference date range, 90-day window, and valid-night count.

Service Map

Service / view model Responsibility
ProtocolBaselineService Freezes and augments the protocol baseline from cached sleep sessions.
ProtocolFormulaAnalysisService Builds per-night snapshots, taken-only rollups, impact summaries, and nightly chart points.
ProtocolFormulaCatalogService Ensures formula catalog versions and selects best version when sufficient data exists.
ProtocolFormulaHomeViewModel Loads active formula, last-night snapshot, baseline, impact, trend snapshots, and tonight save state.
ProtocolAllMetricsViewModel Loads versions, rollups, nightly snapshots, baseline, and active metric chart points.
ProtocolTimelineViewModel Builds version timeline cards from logs and attaches rollup/session data when available.
SyncCoordinator Maintains cached sleep history with 90-day retention and clamps baseline windows to 90 days.
